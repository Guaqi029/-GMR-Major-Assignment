# GMR + OMOMO 创新路线架构说明

## 1. 路线结论

当前最稳且最快的路线是：

**OMOMO 接触阶段感知的 IK 权重自适应 + 轻量时序平滑**

一句话概括：
在不推翻 GMR 原有逐帧 IK 框架的前提下，利用 OMOMO 中天然存在的人-物交互特征，自动识别“接触/操作阶段”，并在这些阶段动态调整上肢、躯干、骨盆等关键部位的 IK 权重，同时对输出关节轨迹做轻量平滑，从而提升操作语义保持、减小抖动，并增强整体稳定性。

这条路线优先于“完整 object-relative retargeting”或“重型动力学优化”的原因是：

1. 改动小，直接嵌入当前 GMR 架构。
2. 和 OMOMO 数据特性强相关，论文故事成立。
3. 不依赖额外训练，不需要新模型，不需要重新设计求解器。
4. 可以先在小样本上快速验证，确认有效后再全量跑数据。

---

## 2. 为什么选这条路线

### 2.1 与当前代码结构匹配

当前 GMR 的核心是：

- 输入每一帧人体关节数据
- 根据 `ik_config` 建立任务目标
- 用 Mink/MuJoCo 做逐帧 IK 求解
- 输出机器人 `qpos`

对应核心文件：

- [motion_retarget.py](/mnt/c/Users/guyiq/Desktop/GMR复现/GMR/general_motion_retargeting/motion_retarget.py:1)
- [smplx_to_g1.json](/mnt/c/Users/guyiq/Desktop/GMR复现/GMR/general_motion_retargeting/ik_configs/smplx_to_g1.json:1)

这个架构最容易插入的增强点不是“换求解器”，而是：

- 改每一阶段的任务权重
- 加阶段相关规则
- 在输出轨迹层做平滑

### 2.2 与 OMOMO 数据特性匹配

OMOMO 不是普通行走数据，而是人-物交互数据。它的重要性在于：

- 含有操作动作
- 含有明显的接触阶段
- 上肢和躯干语义比单纯 locomotion 更关键

如果直接用固定 IK 权重，容易出现：

- 手部操作语义弱化
- 接触阶段手腕/肘部动作不稳定
- 切换阶段抖动增大

所以相比“纯权重调参”，做“接触阶段感知的动态权重”更像一个真正围绕 OMOMO 的创新。

### 2.3 时间成本可控

相比下面两类路线，这条路线更适合当前时间窗口：

不优先做的路线：

- 完整 object-relative hand-object pose retargeting
- dynamics-aware / trajectory optimization / learned retargeter

原因：

- 实现面更大
- 调试成本更高
- 一旦验证不稳定，容易拖慢论文进度

## 3. 论文可表述的创新点

建议将创新点表述为：

**面向人-物交互动作的阶段感知式重定向增强方法**

更具体一点可以写为：

1. 针对 OMOMO 场景，引入基于接触线索的动作阶段划分。
2. 在接触阶段自适应提高上肢与躯干相关 IK 目标权重。
3. 在重定向后引入轻量时序平滑，抑制接触切换附近的高频抖动。
4. 在不引入复杂动力学优化和额外训练的前提下，提升机器人操作动作的稳定性与语义一致性。

这个表述的好处是：

- 不夸张
- 逻辑完整
- 容易通过实验支撑

## 4. 总体架构

### 4.1 基线流程

当前基线：

`SMPL-X / OMOMO motion -> load_smplx_file -> get_smplx_data_offline_fast -> GMR retarget -> qpos -> pkl -> visualization`

### 4.2 改进后流程

改进后建议流程：

`OMOMO pkl -> 解析人体与物体状态 -> 接触阶段检测 -> 阶段感知 IK 权重 retarget -> 输出 qpos -> 轨迹平滑 -> 质量评估 -> 保存`

### 4.3 模块拆分

建议拆成 4 个模块：

1. `OMOMO 数据解析模块`
   目标：从 `.pkl` 中稳定读取人体动作、物体位姿、物体尺度等信息。

2. `接触阶段检测模块`
   目标：根据手部关键点到物体的相对距离，判断当前帧是否处于接触/操作阶段。

3. `阶段感知 IK 模块`
   目标：根据阶段标签调整关键 task 的 position/orientation cost。

4. `时序平滑与评估模块`
   目标：对 `qpos` 或部分关节做轻量平滑，并输出质量指标。

## 5. 具体技术方案

### 5.1 接触阶段检测

先做最小可实现版本，不要一开始就追求复杂接触建模。

#### 输入

- 左右手腕位置
- 左右手肘位置
- 物体中心位姿或物体参考位置

#### 规则

可采用简单阈值法：

- 若任意一只手到物体中心距离小于阈值，则标记为“接触候选”
- 连续若干帧满足条件，则标记为“接触阶段”
- 为避免抖动，可对阶段标签做 1D 滤波或最小持续帧数约束

#### 目的

不是精确物理接触检测，而是得到“操作阶段”标签，用于控制权重切换。

### 5.2 阶段感知 IK 权重

核心思路：

- 非接触阶段：保持基线配置为主
- 接触阶段：提高上肢、手腕、肘部、肩部以及躯干相关权重
- 对腿部和足端权重尽量保持稳定，避免引入新的站姿不稳

建议的调节方向：

- `left_wrist / right_wrist`：位置权重上调
- `left_elbow / right_elbow`：方向或姿态约束适度上调
- `left_shoulder / right_shoulder`：适度上调
- `spine3 / torso / pelvis`：适度上调，保证操作姿态合理

不建议第一版就大改下肢权重，因为最容易引入副作用。

### 5.3 时序平滑

第一版建议只做轻量后处理，不改 IK 内核。

可选方法：

- moving average
- Savitzky-Golay
- exponential smoothing

优先建议：

**只对上肢关节和腰部关节做轻量平滑**

原因：

- 这些部位最容易在接触切换时抖动
- 风险比全身平滑更低
- 不容易破坏步态和支撑

### 5.4 质量评估

为了后续论文和大规模数据筛选，建议同步做简单评估。

推荐采用以下指标，并在实验表中同时标注英文名、中文名和物理含义：

1. `joint_velocity_peak`
   中文：关节速度峰值
   物理含义：机器人所有关节角速度的最大绝对值，反映是否存在瞬时过快动作或尖峰。

2. `joint_jerk_proxy`
   中文：关节抖动代理
   物理含义：关节速度变化率的平均绝对值，近似反映动作是否平滑、是否存在高频抖动。

3. `contact_phase_wrist_jitter`
   中文：接触阶段手腕抖动
   物理含义：仅在接触阶段统计机器人手腕轨迹的高频变化强度，反映操作时手部是否稳定。

4. `foot_sliding_proxy`
   中文：足端滑移代理
   物理含义：脚接近地面时的水平滑动速度均值，反映站立或支撑时脚底是否打滑。

5. `joint_limit_proximity_ratio`
   中文：关节逼近极限比例
   物理含义：关节位置接近机械上下限的时间占比，反映姿态是否逼近不可执行或不自然区域。

6. `human_robot_wrist_consistency`
   中文：人机手腕运动一致性
   物理含义：接触阶段人手腕与机器人手腕运动方向的平均相似度，反映操作语义是否被保留。

7. `contact_ratio`
   中文：接触阶段占比
   物理含义：整段动作中被判定为接触阶段的帧比例，用于描述交互动作的时长占比。

8. `failure_count`
   中文：失败片段计数
   物理含义：人工或规则统计的明显失败样本数量，例如手臂反折、异常抖动、身体姿态失真。

当前实现优先完成前 7 个自动指标，`failure_count` 作为人工审查或后续规则扩展项保留。

## 7. 最小可实现版本

为了保证“先验证、再全量”，建议只做下面这个最小版本：

### Version A

1. 读取 OMOMO 的人体动作
2. 用手腕到物体中心距离检测接触阶段
3. 接触阶段上调手腕、肘部、肩部、躯干权重
4. 对上肢关节和腰部关节做轻量平滑
5. 在 10 到 20 条 OMOMO 样本上做对比

这是最稳的版本，因为：

- 不动底层求解器
- 不需要训练
- 不需要引入复杂接触物理
- 非常容易做 baseline vs improved 对比

## 8. 实验设计

### 8.1 基线

原始 GMR：

- 固定 `smplx_to_g1.json`
- 无接触阶段感知
- 无时序平滑

### 8.2 改进方法

改进版 GMR：

- 接触阶段感知权重调整
- 上肢/躯干轻量平滑

### 8.3 样本选择

建议先从 OMOMO 中挑选：

- 推/拉类动作
- 双手操作类动作
- 转身并操作类动作
- 姿态变化明显的操作类动作

先做 10 到 20 条，不要一开始全量。

### 8.4 对比维度

定性：

- 机器人是否更稳定
- 操作动作是否更像人
- 抖动是否减少

定量：

- 关节速度峰值
- 速度突变次数
- 失败率
- 平均人工评分

## 9. 论文写作建议

### 9.1 题目方向

可以考虑：

**面向人-物交互场景的人形机器人动作重定向增强方法研究**

或：

**基于接触阶段感知的 GMR 人形机器人动作重定向优化**

### 9.2 摘要核心逻辑

摘要建议按这个逻辑写：

1. GMR 能做通用动作重定向，但对 OMOMO 这类交互动作仍存在接触阶段不稳定、上肢语义保持不足等问题。
2. 为此，提出一种接触阶段感知的轻量增强方法。
3. 该方法通过阶段检测、自适应权重调整和轻量时序平滑改善操作动作质量。
4. 在 OMOMO 子集实验中，改进方法在稳定性和语义一致性上优于基线 GMR。

### 9.3 贡献写法

建议写成 3 条：

1. 提出一种面向人-物交互动作的阶段感知式 GMR 增强方法。
2. 设计了无需训练、易于集成的权重自适应与轻量平滑机制。
3. 在 OMOMO 数据上验证了该方法对操作动作稳定性和可用性的提升。
